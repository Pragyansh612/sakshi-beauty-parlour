### Post-Step-9 — Schema fixes, live Supabase project, real gallery photos
A real Supabase project (`.env`, not `.env.local`) was connected for the first time. Running `supabase/schema.sql` against it surfaced two latent bugs that static review hadn't caught (both fixed in `schema.sql` directly):
- `is_admin()` (a `LANGUAGE sql` function) referenced `profiles` before the table existed. Unlike `plpgsql`, Postgres resolves table references in `LANGUAGE sql` function bodies at `CREATE FUNCTION` time, so this failed with `relation "profiles" does not exist` on a fresh run. Fixed by moving the function definition after `CREATE TABLE profiles`.
- `gallery_images.storage_path` had no `UNIQUE` constraint, but `scripts/seed-images.mts` upserts on `onConflict: 'storage_path'` — which requires one. Added `UNIQUE` to the column in `schema.sql`. This project's live table predates the fix; a one-line `ALTER TABLE ... ADD CONSTRAINT ... UNIQUE (storage_path)` is documented in `README.md` for anyone else in the same position.
- Also discovered `scripts/seed-images.ts` and `scripts/seed-slots.ts` had never actually been run end-to-end before: both use top-level `await import('fs')` to read `.env.local`, which fails under `tsx` because the project's `package.json` has no `"type": "module"` (so `.ts` files compile as CJS, and CJS can't have top-level await). Renamed both to `.mts` (forces ESM regardless of `package.json`) and made the env-loading fall back to `.env` if `.env.local` doesn't exist.
- Created the `gallery` (public) and `avatars` (private) Storage buckets on the live project (per `README.md`'s setup checklist — they didn't exist yet).
- Converted all 20 `.HEIC` files in `./acheivement/` to `.jpg` using a new `scripts/convert-heic.mts` (via the `heic-convert` package — no system `ImageMagick` available in this environment, which is what the original seed script's warning message assumed). Originals kept; `.jpg` siblings added and committed.
- Ran the (fixed) seed script for real: all 8 `work/` + 21 `acheivement/` photos uploaded to Supabase Storage and inserted into `gallery_images` (29 rows total).
